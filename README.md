# 교회 주보 편집기 GitHub 배포 및 사용 안내

안녕하세요! 이 안내서는 '라온동행교회 주보 편집기' 프로그램을 GitHub에 올려서 누구나 접속할 수 있는 실제 웹사이트로 만드는 방법을 설명합니다.

컴퓨터에 익숙하지 않은 분들도 따라 하실 수 있도록 최대한 쉽게 작성되었습니다.

---

## 왜 GitHub를 사용하나요?

-   **무료 호스팅**: GitHub Pages라는 기능을 통해 별도 비용 없이 웹사이트를 인터넷에 게시할 수 있습니다.
-   **안전한 코드 보관**: 작성된 코드를 온라인에 안전하게 저장하고 관리할 수 있습니다.
-   **쉬운 공유**: `https://...` 로 시작하는 나만의 주보 편집기 웹사이트 주소가 생겨, 언제 어디서든 접속하고 다른 사람에게 공유하기 편리합니다.

---

## 배포 방법 (총 4단계)

### 1단계: GitHub 저장소(Repository) 만들기

먼저 코드를 올려둘 온라인 공간을 만듭니다.

1.  [GitHub](https://github.com)에 접속하여 로그인합니다. (계정이 없다면 회원가입이 필요합니다.)
2.  화면 오른쪽 위 **+** 아이콘을 누르고 **New repository**를 선택합니다.
3.  **Repository name** (저장소 이름) 란에 원하는 이름을 영어로 입력합니다. (예: `church-bulletin`, `raon-church-weekly`)
4.  저장소 공개 설정을 **Public**으로 선택해야 GitHub Pages를 무료로 사용할 수 있습니다.
5.  다른 설정은 건드리지 않고, 맨 아래 **Create repository** 버튼을 클릭합니다.

### 2단계: 프로젝트 파일 업로드하기

방금 만든 저장소에 현재 가지고 있는 모든 파일을 올립니다.

1.  저장소가 만들어지면 보이는 화면에서 **uploading an existing file** 링크를 클릭합니다.
2.  탐색기가 열리면 가지고 있는 프로젝트 파일 전체(`index.html`, `index.tsx`, `App.tsx`, `types.ts` 및 `components` 폴더 등)를 끌어다 놓거나 선택해서 업로드합니다.
3.  파일이 모두 올라간 것을 확인한 후, 아래쪽 **Commit changes** 버튼을 클릭하여 저장합니다.

### 3단계: GitHub Pages 기능 활성화하기

업로드한 파일을 실제 웹사이트로 발행하는 과정입니다.

1.  저장소 페이지 상단 메뉴에서 **Settings** 탭을 클릭합니다.
2.  왼쪽 메뉴에서 **Pages**를 클릭합니다.
3.  `Build and deployment` 항목에서, `Source`를 **Deploy from a branch**로 설정합니다.
4.  `Branch` 항목에서 `main` 브랜치를 선택하고, 폴더는 `/ (root)`로 둔 다음 **Save** 버튼을 누릅니다.

### 4단계: 웹사이트 주소 확인하기

1.  **Save** 버튼을 누르고 잠시(1~5분) 기다린 후 새로고침하면, "Your site is live at..." 라는 메시지와 함께 웹사이트 주소가 나타납니다.
2.  이 주소(예: `https://<나의-GitHub-아이디>.github.io/<저장소-이름>`)가 바로 **나만의 주보 편집기 링크**입니다! 이 주소를 즐겨찾기 해두고 사용하시면 됩니다.

---

## 배포 후 사용 방법

1.  **주보 편집**: 위에서 얻은 나만의 웹사이트 주소로 접속하면 익숙한 '수정 모드' 화면이 나타납니다. 여기서 주보 내용을 자유롭게 수정합니다.
2.  **공유하기**: 수정이 끝나면, 오른쪽 아래 **파란색 공유 버튼**을 누릅니다.
3.  **링크 복사 및 전달**: "링크가 복사되었습니다!" 메시지가 뜨면, 카카오톡이나 문자로 성도님들께 붙여넣기 하여 전달합니다. 성도님들은 수정 화면 없이 완성된 '보기 모드'의 주보를 보게 됩니다.

**중요**: 이 프로그램은 데이터를 GitHub에 저장하는 방식이 아닌, **공유 링크 자체에 모든 데이터를 담아 보내는 방식**으로 작동합니다. 따라서 주보를 수정할 때마다 매번 새로운 공유 링크를 만들어 전달해주셔야 합니다.
